The table below includes performance data on rendering the standard procedural 
database (SPD) scenes and some widely used Stanford 3D Scanning Repository 
scenes. The SPD is described in `A proposal for standard graphics 
environments'' by Eric A. Haines and has an associated web page:
      www.acm.org/pubs/tog/resources/SPD
The measures of performance used are from "Comparison methodology for ray 
shooting algorithms." by V.Havran and W. Purgathofer (Tech. report 
TR-186-2-00-20, Vienna Univ. of Technology, Nov. 2000)
The data includes and individual scenes values for the key performance 
parameters of the ten SPD scenes of groups G4,  G5 and G6 (the SPD scenes 
of roughly ten thousand, one hundred thousand, and one million primitives).


In the ray tracing experiments the rendering is strictly done according 
to the rules stated in the SPD  Readme.txt file (e.g. 513 x 513 primary 
rays, plus an additional four levels of reflection). Isolation algorithms 
use the ``classic settings'' for wfac and minNForIsolation though these 
settings  are know to be sub-optimal. Also, FORT does not use primitive 
caching, ray coherence methods, nor the shadow buffer or light buffer 
algorithms.  FORT does not perform explicit ray-box intersection tests
prior to the ray-primitive intersection test. The exception are that the 
the ray-box test are evaluated prior to the intersection of rays with
large polygons (used in the gears scenarios); and also for the RSA listed 
as ``DET(BB)'' which performs a ray-box test prior to every ray-primitive 
test. 

The DE-tree nodes are 24 bytes in size for all the presented results.
With Np being the number of graphics primitives, there are Np -- 1 interior 
tree nodes and Np leaf nodes used. Each leaf node has a direct pointer to a 
graphics primitive (field obj). Algorithm  SMI is used such that the first
dimension is the ``X'' axis (d=0), and the dimension in the next level of 
recursion is (d+1)%3. The DE-tree search algorithm is called with initial 
arguments mint and maxt - the initial minimum and maximum ray-object distance 
limits - to be the same values for all shadow rays, the  same values for all 
primary rays, the  same values for all other rays. 

The FORT experiment data is collected on a  Pentium 4 of 1.8 GHz, 400 MHz
FSB,  256 MB RAM, 512K L2 cache, and using SUSE Linux 8.2. The compiler 
is gcc 3.4.3  using options  ``-O3 -mfpmath=sse -msse2 -m32''. 

Symbols for the Table.
Tb -- the time (in seconds) to build the RSA.
Tr -- the time (in seconds) to perform all ray tracing 
      calculations once the RSA is created.
Np -- the number of graphics primitives in the scene.
Nr -- the number or rays used in rendering.
Nts -- the average number of nodes accessed per ray.
Ng -- the number of generic nodes in the RSA
       (usually the number of interior nodes for tree data structures)
Ne -- the maximum number of elementary nodes in the RSA data structure
      (usually the number of leaf nodes for tree data structures).
Ner -- the total number of references to objects in elementary 
      nodes of the RSA DS.
Ritm -- the ratio of ray-object intersection tests performed to the minimum 
      number of intersection tests.
Nit -- the average number of ray-object intersection tests performed per ray.
RSA/PS - the RSA and the partitioning strategy.
KDT, DET, BVH -- the kd-tree, DE-tree, and BVH RSAs.
OM, OMI, SM, SMI, MSM -- the partitioning strategies respectively called 
     the object median, object median plus isolation, spatial median,
      spatial median plus isolation, and Mansfield-Shirley-Morley.
